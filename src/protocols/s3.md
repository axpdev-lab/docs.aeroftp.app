# S3-Compatible Storage

AeroFTP supports Amazon S3 and any S3-compatible object storage service. This includes major providers like Wasabi, Backblaze B2, Cloudflare R2, DigitalOcean Spaces, Storj, MinIO, and more.

## Connection Settings

| Field | Value | Notes |
|-------|-------|-------|
| Endpoint | Service endpoint URL | Auto-filled when using a preset |
| Region | AWS region or equivalent | e.g. `us-east-1`, `eu-west-1` |
| Access Key | Your access key ID | From the provider's console |
| Secret Key | Your secret access key | Stored encrypted in the OS keyring |
| Bucket | Target bucket name | Must already exist |

## Presets

| Provider | Endpoint Template | Region Selection | Notes |

### Cloudflare R2

R2 requires your **Cloudflare Account ID** instead of a region. AeroFTP provides a dedicated Account ID field. The endpoint is computed automatically:

```
{your-account-id}.r2.cloudflarestorage.com
```

Generate R2 API tokens in the Cloudflare dashboard under **R2 > Manage R2 API Tokens**.

## Features

- **Path-Style Addressing**: AeroFTP uses path-style URLs (`endpoint/bucket/key`) by default, which is compatible with all S3 providers including MinIO.
- **Multipart Upload**: Large files are uploaded in parts for reliability and resumability.
- **Pagination**: Continuation token-based listing handles buckets with millions of objects.
- **Pre-signed URLs**: Generate temporary download links for sharing.

## Tips

- For **MinIO**, set the endpoint to your MinIO server URL (e.g. `http://minio.local:9000`) and use your MinIO access/secret keys.
- If you get `SignatureDoesNotMatch` errors, verify that your system clock is accurate. S3 signatures are time-sensitive.
- When editing a saved S3 connection, the endpoint is auto-resolved from the registry if it was not stored previously (backward compatibility).
- S3 does not have a traditional directory structure. AeroFTP emulates folders using `/` prefix delimiters.
